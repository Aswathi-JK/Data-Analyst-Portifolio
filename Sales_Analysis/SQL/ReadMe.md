# 📊 Sales Data Analysis using SQL

## 📌 Objective

Analyze sales data using SQL to identify business insights, top-performing regions, profitable product categories, and loss-making products.

---

## 📂 Dataset

The project uses a sales dataset in CSV format containing information such as:

* Sales
* Profit
* Region
* Product Category
* Product Name

---

## 🛠️ Tools Used

* MySQL Workbench
* SQL
* CSV Files
* Git & GitHub

---

## SQL Concepts Used
* SELECT
* WHERE
* GROUP BY
* ORDER BY
* LIMIT
* SUM()
* AVG()
* COUNT()
* ROUND()

## 🔍 Analysis Performed

### 1. Sales Analysis

* Calculated Total Sales
* Calculated Total Profit
* Calculated Total Orders

### 2. Regional Performance Analysis

* Aggregated sales by region
* Identified the top-performing region

### 3. Product Category Analysis

* Aggregated sales by product category
* Identified the highest revenue-generating category
* Calculated average sales by category

### 4. Profit Analysis

* Calculated profit by region


### 5. Top Products

Identified the highest-selling products.

### 6. Loss Analysis

* Identified products with negative profit
* Extracted loss-making products for further investigation

## 📈 Key Insights

* West generated the highest sales revenue.
* Technology was the highest-performing product category.
* Ontario was among the top-performing regions.
* Several products generated negative profit and require further investigation.

## 🎯 Conclusion

This project demonstrates the use of SQL for data analysis, data aggregation, business insight generation, and reporting. The analysis helps identify sales performance trends, profitable categories, and loss-making products that can support data-driven business decisions.
